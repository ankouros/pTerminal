# Change Checklist (Template)

Use this checklist for every change or feature.
Copy this file into your PR/commit notes or issue description and tick items.

## Change Summary

- [ ] What changed (1-3 bullets)
- [ ] Why it changed (1-2 bullets)
- [ ] User impact noted

## Docs Updates

- [ ] `docs/usage.md` updated if user-facing behavior changed
- [ ] `docs/onboarding.md` updated if dev workflow changed
- [ ] `docs/lan-teams.md` updated if LAN sync behavior changed
- [ ] `docs/README.md` updated if docs structure changed

## Contracts / Roadmap / Issues

- [ ] `CONTRACTS.md` updated if expectations changed
- [ ] `ROADMAP.md` updated (current vs planned)
- [ ] `issues-tracker.md` updated (new issue/proposal/status)

## Tests and Validation

- [ ] Tests added/updated or explicitly deferred (reason noted)
- [ ] `go test ./...` run (or reason noted)
